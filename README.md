# 🔍 Network Scanner

A local network device scanner written in Python. Automatically detects all active hosts on your network and displays **IP**, **MAC**, **hostname**, and **vendor** in a colorful terminal table.

## ✨ Features

- Automatic detection of the local network range (`/24`)
- Two scanning modes:
  - **ARP/Scapy** (accurate, returns real MAC) — requires root/sudo
  - **Ping** (fallback without privileges) — works without root
- Hostname resolution via reverse DNS
- Vendor identification from the MAC OUI
- Formatted output with [`rich`](https://github.com/Textualize/rich) (tables, colors, spinner)
- Concurrent scanning with `ThreadPoolExecutor`

## 📦 Requirements

- Python 3.8+
- Linux/macOS (the `run.sh` is a shell script; on Windows, run `python network_scanner.py` directly)

Dependencies (in `requirements.txt`):

```
scapy>=2.5.0
mac-vendor-lookup>=0.1.12
rich>=13.0.0
```

## 🚀 Installation

```bash
git clone <repo-url>
cd network-scan
chmod +x deploy.sh
./deploy.sh
```

The `deploy.sh` script:

1. Verifies Python 3 is installed
2. Creates a virtualenv in `.venv/`
3. Installs the dependencies
4. Generates the `run.sh` wrapper

## 🖥️ Usage

```bash
sudo ./run.sh                    # full mode (ARP + MAC)
./run.sh                         # ping mode (no root, no MAC)
./run.sh 192.168.0.0/24          # manual range
```

Or directly:

```bash
sudo python network_scanner.py
python network_scanner.py 10.0.0.0/24
```

## ⚠️ Notes

- Without `sudo`, Scapy cannot send ARP packets — the script automatically falls back to ping mode (no MAC).
- ARP scanning only works within the same subnet (L2 broadcast).
- Some devices may not respond to ping even when active (firewall).
